HW 4

In homework 4, you are required to implement a simple calculator called StackCalculator using LinkedStack. StackCalculator receives an infix arithmetic expression and returns the result of the expression. The calculation is done by two steps, changing infix expression to postfix expression (changeToPostfix), and evaluating postfix expression to achieve the result (evaluate).

To simplify the problem, we set some limitations (rules) on StackCalculator.

  1. StackCalculator only deals with four basic operators, *+, -, , / and three kinds of brackets, which are (, ), {,}, [,]. The operation order is described as below.
  • *, / -> +,-
    • Multiplication and division should be operated before addition and subtraction.
  • () -> {} -> []
    • Equations in parentheses should be calculated before equations in curly brackets.
    • Equations in curly brackets should be calculated before equations in square brackets.

  1. StackCalculator can receive the input expression consists of the operators, three kinds of brackets, and any real number. Each component will be separated with a space, e.g. “3.25 * 12 – 8 + -7”. The components of converted postfix expression should be separated with a space too.

  2. StackCalculator cannot divide the number by zero. Therefore, if such expression (input or intermediate) is given, StackCalculator throws the DividedByZeroException. StackCalculator cannot calculate the input expression with unbalanced parenthesis. Therefore, if such expression is given, StackCalculator throws the UnbalancedParenthesisException.

  3. StackCalculator utilizes LinkedStack, not java.util.Stack.

The claims caused by the problem that is out of the limitations (rules) will not be allowed.

Submit the implementation of the StackCalculator class through Elice system. You have to implement two methods (changeToPostfix and evaluate) in StackCalculator.java file in the Elice system. Please do not change the parameters, modifiers, and exceptions in the method.
